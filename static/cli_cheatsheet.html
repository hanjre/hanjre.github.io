## Terminal Basics & Shells
**Shells:** macOS (zsh), Linux (bash), Windows (PowerShell, plus WSL for Linux shell).
**Where am I?** `echo $SHELL` (Mac/Linux) • `$PSVersionTable` (PowerShell).

```bash
# Open Terminal
# macOS: Spotlight → "Terminal"
# Windows: Start → "PowerShell" (or "Terminal"), for Linux tools install WSL.
```

**Tip:** Your prompt often shows `user@host folder %` → you’re inside `folder` on your local machine.

## Navigation & Paths
- `pwd` – print working directory
- `ls` / `dir` – list directory contents
- `cd path` – change directory
- `cd ..` – up one level • `cd ~` – go to home
- Absolute path starts with `/` (Mac/Linux) or `C:\` (Windows).

```bash
pwd
ls -lah # long list, human sizes, show hidden
cd ~/Projects # go to Projects (macOS/Linux)
cd "C:\Users\YOU" # Windows (PowerShell)
```

## Files & Folders
Create, copy, move, remove; **be careful with rm**.

```bash
# Create
mkdir -p data/raw data/processed # -p creates parents
touch notes.md # new empty file

# Copy / Move
cp README.md docs/README.md
mv notes.md docs/notes.md

# Delete (safe)
rm file.tmp
rm -r old_folder # recursive (CAUTION)
# Safer on macOS/Linux: send to Trash with 'trash' (brew install trash)
```

Windows PowerShell:
```powershell
New-Item notes.md -ItemType File
New-Item data\raw -ItemType Directory
Copy-Item README.md docs\
Move-Item notes.md docs\
Remove-Item old_folder -Recurse # CAUTION
```

## Search & Find
Find files by name/content. Grep is your friend.

```bash
# Find by name
find . -iname "*report*.csv"

# Search text inside files (recursive)
grep -RIn "SELECT .* FROM" .

# ripgrep (faster): brew install ripgrep | choco install ripgrep | apt install ripgrep
rg -n "TODO" .
```

PowerShell equivalents:
```powershell
# By name
Get-ChildItem -Recurse -Filter *report*.csv

# By content
Select-String -Path .\* -Pattern "SELECT .* FROM" -CaseSensitive:$false
```

## Viewing & Editing Files
```bash
# Quick looks
head -n 20 big.csv
tail -f app.log # live follow
less big.csv # page through (q to quit)

# Simple edits (nano is beginner-friendly)
nano config.env
vi config.env # vim (advanced)

# macOS quick look from terminal (preview window)
qlmanage -p file.png 2>/dev/null
```

PowerShell:
```powershell
Get-Content .\app.log -Wait # like tail -f
notepad .\config.env
```

## Permissions & Ownership
```bash
# See permissions
ls -l

# Make a script executable
chmod +x run.sh

# Change owner (need sudo)
sudo chown -R $USER:$USER ./project
```

PowerShell:
```powershell
# Windows uses ACLs; basic unblock for downloaded scripts:
Unblock-File .\script.ps1
```

## Processes & System Info
```bash
ps aux | head
top # live view (q to quit)
htop # nicer top (brew/choco/apt install htop)
kill -9 <PID> # force kill
```

PowerShell:
```powershell
Get-Process | Select-Object -First 10
Stop-Process -Id <PID> -Force
```

## Networking & Localhost
```bash
# What’s listening?
lsof -i :5432 # Postgres port
netstat -an | head

# Test connectivity
ping 8.8.8.8
curl http://localhost:8000
```

PowerShell:
```powershell
Get-NetTCPConnection -LocalPort 5432
Test-NetConnection -ComputerName localhost -Port 8000
Invoke-WebRequest http://localhost:8000
```

## Environment & Packages
**macOS/Linux package managers:** Homebrew (`brew`), APT (`apt`), dnf/yum.
**Windows:** winget, Chocolatey (`choco`).

```bash
# Homebrew (macOS/Linux)
/bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"
brew update && brew upgrade
brew install git python node docker htop ripgrep

# APT (Ubuntu/Debian)
sudo apt update && sudo apt upgrade -y
sudo apt install -y git python3 python3-venv docker.io ripgrep

# Windows (winget)
winget install Git.Git Python.Python.3 NodeJS.NodeJS Docker.DockerDesktop BurntSushi.ripgrep.MSVC
```

## Python & Virtual Environments
Keep dependencies isolated per project.

```bash
# Create venv (per project)
cd ~/Projects/30_Data/Active/2025_02_Data_Quality_Audit
python3 -m venv .venv
source .venv/bin/activate # macOS/Linux
# Windows PowerShell:
# .venv\Scripts\Activate.ps1

pip install --upgrade pip
pip install pandas jupyterlab numpy pyarrow

# Freeze deps
pip freeze > requirements.txt

# Deactivate
deactivate
```

Jupyter quick start:
```bash
python -m pip install jupyterlab
jupyter lab
```

## Git & GitHub Workflow
Initialize, branch, commit, push (feature-branch model).

```bash
# First-time global config
git config --global user.name "Jeremy Harris"
git config --global user.email "you@example.com"
git config --global init.defaultBranch main

# New repo
cd ~/Projects/30_Data/Active/2025_02_Data_Quality_Audit
git init
cp ~/Projects/00_ADMIN/Git_Configs/python.gitignore .gitignore
git add .
git commit -m "init: scaffold project"

# Create feature branch
git checkout -b feat/data-cleaning

# After work
git add -A
git commit -m "feat: add cleaning pipeline"
git push -u origin feat/data-cleaning # after adding remote
```

Add remote & push initial:
```bash
git remote add origin https://github.com/<user>/<repo>.git
git push -u origin main
```

Update main with latest:
```bash
git checkout main
git pull --rebase origin main
git merge --no-ff feat/data-cleaning
git push origin main
```

## Docker Essentials
Containers for databases, dev services.

```bash
# Verify
docker --version
docker ps

# Run Postgres locally
docker run --name pg -e POSTGRES_PASSWORD=postgres -p 5432:5432 -d postgres:16

# Logs & exec
docker logs -f pg
docker exec -it pg psql -U postgres

# Stop/remove
docker stop pg && docker rm pg
```

docker-compose.yml example:
```yaml
version: "3.9"
services:
db:
image: postgres:16
restart: unless-stopped
environment:
POSTGRES_PASSWORD: postgres
ports:
- "5432:5432"
volumes:
- ./volumes/pg:/var/lib/postgresql/data
```

Run:
```bash
docker compose up -d
docker compose logs -f db
```

## Databases (Postgres & SQLite)
```bash
# Postgres CLI
psql -h localhost -U postgres -d postgres
# Inside psql:
-- \l list databases
-- \dt list tables
-- \q quit

# SQLite quick ops
sqlite3 my.db "SELECT sqlite_version();"
sqlite3 my.db ".tables"
sqlite3 my.db "CREATE TABLE t(id INTEGER PRIMARY KEY, name TEXT);"
```

## macOS · Windows · Linux Equivalents

| Task | macOS/Linux | Windows (PowerShell) |
|-----|-------------|----------------------|
| List files | `ls -lah` | `Get-ChildItem -Force` |
| Current dir | `pwd` | `Get-Location` |
| Move file | `mv a b/` | `Move-Item a b\` |
| Copy file | `cp a b/` | `Copy-Item a b\` |
| Delete dir | `rm -r dir` | `Remove-Item dir -Recurse` |
| Tail file | `tail -f app.log` | `Get-Content app.log -Wait` |
| Find text | `grep -RIn PATTERN .` | `Select-String -Path .\* -Pattern PATTERN` |
| Process list | `top` / `htop` | `Get-Process` |
| Kill PID | `kill -9 PID` | `Stop-Process -Id PID -Force` |

## Safety Nets
- **Ctrl + C** → cancel a stuck command.
- **Up/Down arrows** → cycle command history.
- **Tab** → autocomplete paths/commands.
- `trash` CLI moves to Trash instead of permanent delete (macOS: `brew install trash`).
- Use `.gitignore` to avoid committing large data, envs, secrets.
- Keep active work **local** (not iCloud): `~/Projects/...`
